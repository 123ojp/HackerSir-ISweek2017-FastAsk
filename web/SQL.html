SQL注入介紹

SQL是一個資料庫語言，而注入可以說是駭入。

SQL注入（SQL injection），是發生於應用程式之資料庫層的安全漏洞。簡而言之，是在輸入的字串之中夾帶SQL指令，在設計不良的程式當中忽略了檢查，那麼這些夾帶進去的指令就會被資料庫伺服器誤認為是正常的SQL指令而執行，因此遭到破壞或是入侵。

可注入範例網站


說明

這個網站的登入驗證的SQL查詢代碼為

strSQL = "SELECT * FROM users WHERE (username = ' (userName變數) ') and (password = (passWord變數)');"

在username填入管理者帳號

admin

密碼部分則是惡意塞入

' OR '1'='1';

將導致原本的SQL字串被填為

strSQL = "SELECT * FROM users WHERE (username = 'admin') and (password ='' OR '1'='1';');"

也就是實際上運行的SQL命令會變成下面這樣的

strSQL = "SELECT * FROM users WHERE username = 'admin';"

因此達到無密碼，可登入網站。
